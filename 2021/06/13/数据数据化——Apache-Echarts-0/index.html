基本概念总结

快速总结一下官方文档中的基本概念要点

系列（series）

series指定图表的类型。如图，图中series 0对应的type参数为pie，代表饼图，类似的系列类型还有：line（折线图）、bar（柱状图）scatter（散点图）、graph（关系图）、tree（树图）、…

组件（component）

组件描述了图表中的信息，echarts 中的常见组件：xAxis（直角坐标系 X 轴）、yAxis（直角坐标系 Y 轴）、grid（直角坐标系底板）、angleAxis（极坐标系角度轴）、radiusAxis（极坐标系半径轴）、polar（极坐标系底板）、geo（地理坐标系）、dataZoom（数据区缩放组件）、visualMap（视觉映射组件）、tooltip（提示框组件）、toolbox（工具栏组件）、series（系列）、…

option

所有的图表信息都存在option中， option 是个大的 JavaScript 对象

不推荐这么写数据（不方便传参），实际使用过程中建议使用dataset：

组件定位

采用绝对定位，基于 echarts 容器 的DOM 节点。

取值：

  • 绝对数值（例如 bottom: 54 表示：距离 echarts 容器底边界 54 像素）
  • 基于 echarts 容器高宽的百分比（例如 right: '20%' 表示：距离 echarts 容器右边界的距离是 echarts 容器宽度的 20%）
